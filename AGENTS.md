# Règles permanentes pour Codex

- Répondre en français et expliquer les modifications en français.
- Travailler une seule grande étape à la fois.
- Respecter le cahier des charges du projet.
- Rester cohérent avec les cours Studi.
- Utiliser HTML, Sass, Bootstrap et JavaScript Vanilla.
- Ne jamais utiliser Angular, React ou Vue.
- Ne pas installer de dépendance sans l’accord explicite de l’utilisatrice.
- Ne pas créer de fonctionnalité non demandée.
- Ne pas modifier de fichier sans rapport avec la tâche en cours.
- Indiquer les fichiers créés ou modifiés.
- Expliquer les commandes exécutées.
- Tester les modifications lorsque cela devient possible.
- Ne jamais faire de commit ou de push Git sans l’accord explicite de l’utilisatrice.
- Privilégier une solution simple, lisible et compréhensible pour une débutante.
